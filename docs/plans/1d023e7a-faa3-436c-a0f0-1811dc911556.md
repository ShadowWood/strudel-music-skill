## Goal
Create a standalone one-page static HTML site under `pages/` that introduces the `strudel-music-skill` project to visitors, deployable via GitHub Pages from the `pages/` directory.

## Context
- Repo: `github.com/ShadowWood/strudel-music-skill.git` (single `main` branch)
- Project is a Claude Code skill that translates natural-language music descriptions into Strudel code played in a browser via Vite + strudel.cc iframe
- Existing content: `README.md`, `SKILL.md`, `reference.md`, `genres.md`, `examples.md`
- `docs/` already exists for planning artifacts — this is NOT the deployment target
- The new site goes into `pages/` (new top-level dir), keeping it separate from `templates/` (Vite player) and `docs/`
- `pages/` is not in `.gitignore`, so git will track it automatically
- No build tool, bundler, or npm dependency — pure HTML + CSS (works on any static host including GitHub Pages)

## Assumptions
- The site will be served from the `pages/` directory root (GitHub Pages can be configured at repo settings to serve from `/pages`, or content can be pushed to a `gh-pages` branch — plan covers the directory structure; repo admin configures the Pages source)
- All text is English (matching existing project convention)
- The site does NOT embed the live Strudel REPL or Vite player — it's a marketing/landing page only
- The hand-drawn / sketch aesthetic is achieved via pure CSS (custom pseudo-elements, inline SVG icons, border treatments) — no external image assets

## Implementation Notes

### File creation
Only one file to create: `pages/index.html` — self-contained HTML document (~200-300 lines).

### Design breakdown — hand-drawn / sketch style via pure CSS

1. **Sketch underlines on headings**: `::after` pseudo-elements with dashed border-bottom + slight `rotate(-0.5deg)` and uneven `border-radius` to simulate human-drawn lines.
2. **Hand-drawn card borders**: `border: 2px solid` with uneven `border-radius` values (e.g. `10px 12px 8px 14px`) + subtle offset `box-shadow`.
3. **Sketchy SVG icons**: Inline simple SVG paths for feature cards (music note, code brackets, play triangle) using `stroke-linejoin: round` and `stroke-linecap: round`.
4. **Color palette** (from existing player shell):
   - Background: `#0e0f12`
   - Surface / card bg: `#15171c`
   - Borders: `#2a2d35`
   - Text primary: `#e7e9ee`
   - Text secondary: `#b3b7c0`
   - Accent: `#4caf50`
   - Accent hover: `#66bb6a`
5. **Typography**: `font-family: ui-sans-serif, system-ui, -apple-system, sans-serif`
6. **Layout**: Single-column centered layout with max-width container (~800px). Sections: Hero → About → Features → Genres → Quick-start → Footer.

### Sequence

### Risk controls

- Fallback: if hand-drawn aesthetic proves unsatisfying with pure CSS, revert to clean minimal (Anthropic-style) without sketch elements; CSS can be tuned iteratively
